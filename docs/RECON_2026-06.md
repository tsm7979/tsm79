## Target architecture (this phase)

Per the new direction: **TSM is a deterministic trust membrane around AI, not an
AI.** AI is an untrusted upstream compute endpoint. The live gate is rules,
crypto, state, and human governance.

Bidirectional membrane:

```
Client → INGRESS (authn/mTLS, tenant bind, rate limit, normalize)
       → deterministic pipeline (normalize → signatures → structural →
         entropy → context rules → policy engine → decision)
       → {block | redact | route_local | quarantine | allow}
       → AI provider / local model / human queue
       → EGRESS (response scan: secret echo, PII leak, injection-back,
         cross-tenant) → audit-sign → Client
```

Seven deterministic layers to complete/harden:
1. Signature detection (regex / structural / checksum) — exists, harden
2. Entropy analysis — exists, harden
3. Policy engine (signed DSL → IR → dataplane) — partial
4. Rule-based content classification (jailbreak phrase families + normalization) — partial
5. Reputation DB (hashed known-bad) — **missing**
6. Human quarantine (verdict exists; review API missing — GH #38)
7. Sovereign overlay security (Ed25519 / signed records / Kademlia / TLS / Merkle / RBAC) — slice 1+2 done

Plus the missing **egress plane** (response inspection) — the highest-leverage gap.
