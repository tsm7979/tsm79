## Methodology

- Inline detection primitive per iteration: `PIIDetector().scan(text)`.
- Timing via `time.perf_counter_ns()` around that call only.
- 300 warm-up iterations discarded (import / regex compile / cold cache excluded).
- Percentiles are nearest-rank over the full sample set.
- Single-threaded by design; multiply by core count for a first-order throughput estimate.
- Severity column proves real work: clean->none, pii/secret/mixed->critical.

## Honest caveats

- Python reference implementation. The Rust dataplane targets lower latency;
  those numbers are design targets until its build is verified (GitHub #36).
- Numbers vary with hardware. Run it on your own box.
- This is *added* latency. Upstream model latency is unaffected and not shown.
